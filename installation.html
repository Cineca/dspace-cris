Installation instructions

DSpace CRIS is an addon for the DSpace platform. This page describes as install the addon on the top of a customized version of DSpace JSPUI 1.8.2 that already includes the "core" changes needed to integrate DSpace CRIS.

The source code of the customized pre-integrated version of DSpace 1.8.2 can be found in the CILEA GitHub Repository

  • current line of development: https://github.com/CILEA/DSpace/tree/dspace-1_8_x-cris
  • tagged release:
    • dspace-cris-1.8.2.0-beta : https://github.com/CILEA/DSpace/tree/dspace-cris-1.8.2.0-beta

Installation steps

Be sure to understand the general DSpace installation procedure before to attempt to use this instructions: https://wiki.duraspace.org/display/DSDOC18/Installation

A complete, step by step, procedure that include also steps from the standard dspace installation is available at the following page: Go to step-by-step

  1. Pre-Installation: install required artifacts in maven local repository
  2. Checkout the source of the pre-integrated DSpace 1.8.2 version 
    
$ git clone https://github.com/CILEA/DSpace.git --branch dspace-cris-1.8.2.0-beta dspace-parent/

usercris@servercris /e/crisinstallation

$ cd dspace-parent/

usercris@servercris /e/crisinstallation/dspace-parent (dspace-1_8_x-cris)

$ git submodule sync

usercris@servercris /e/crisinstallation/dspace-parent (dspace-1_8_x-cris)

$ git submodule update --init dspace-cris/

usercris@servercris /e/crisinstallation/dspace-parent (dspace-1_8_x-cris)
  3. Initial Configuration:

    Edit [dspace-source]/dspace/config/dspace.cfg, in particular you'll need to set these properties: 

    
dspace.dir - must be set to the dspace directory.

dspace.url - complete URL of this server's DSpace home page.

dspace.hostname - fully-qualified domain name of web server.

dspace.name - "Proper" name of your server, e.g. "My Digital Library".

db.password - the database password you entered in the previous step.

mail.server - fully-qualified domain name of your outgoing mail server.

mail.from.address - the "From:" address to put on email sent by DSpace.

feedback.recipient - mailbox for feedback mail.

mail.admin - mailbox for DSpace site administrator.

alert.recipient - mailbox for server errors/alerts (not essential but very useful!)

registration.notify - mailbox for emails when new users register (optional)
  4. Installation Package: 
    
usercris@servercris /e/crisinstallation

$ cd dspace-parent/

usercris@servercris /e/crisinstallation/dspace-parent

$ mvn package

  5. Build DSpace and Initialize Database 
    
cd [dspace-source]/dspace/target/dspace-[version]-build

ant fresh_install`
  6. Create CRIS folders: 
    
usercris@servercris [dspace-source]/dspace/target/dspace-[version]-build

$ cd [dspace]

usercris@servercris [dspace]

$ mkdir rp-files

$ mkdir rg-files

$ mkdir ou-files`
  7. Update database to CRIS: run the files sql that generate update 
    [dspace]/etc/postgres/addon-*.sql
  8. Deploy the Web Applications 
    
cp -R [dspace]/webapps/dspace/jspui* [tomcat]/webapps/
  9. Deploy the Solr 4 
    
Download from http://lucene.apache.org/solr/mirrors-solr-latest-redir.html? and copy webapp downloaded under [tomcat]/webapps
Edit the env-entry tag onto the web.xml file to point at [dspace]/solr

Post installation

Now the DSpace-CRIS module is setup and running but the configuration is empty, you can populate the database with a basic configuration running the base-configuration-crismodule.sql: 

[dspace]/etc/postgres/base-configuration-crismodule.sql

or you can use the aministrative UI of the CRIS module to setup your configuration.

The base-configuration-crismodule.sql file create for you a base metadata configuration for Researcher profile, Organization Unit and Project entity. You can alter this configuration from the administration section of the CRIS module.

Login with your administrator account and go to "Administer->CRIS Module".
